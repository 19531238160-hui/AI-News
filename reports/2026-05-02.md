## 今日速览

今天的 AI 新闻有一条非常清晰的主线：**从“做出效果”转向“稳定上线”**。  
对 AI 训练师初学者来说，最值得关注的是三件事：

1. **评估优先**：多篇内容都在讲 EDD（评估驱动开发）、Agent 评测清单、OpenEvals。
2. **模型可迁移**：企业正在准备跨模型切换，避免被单一模型/厂商绑定。
3. **真实业务落地**：身份核验、政府级部署等案例说明“数据流程 + 评估流程”比“单次 Prompt 调参”更重要。

## 重点新闻

## 1) AWS 发布 LLM 迁移方法论（生产环境模型切换）
- **来源**：Planet AI Aggregate（AWS）
- **链接**：https://aws.amazon.com/blogs/machine-learning/aws-generative-ai-model-agility-solution-a-comprehensive-guide-to-migrating-llms-for-generative-ai-production/
- **发生了什么**：AWS 给出一套在生产环境迁移/升级 LLM 的系统框架，包括提示词转换、优化和迁移流程。
- **为什么重要**：企业不会永远只用一个模型。训练师要学会“**提示词可移植性**”和“**跨模型一致评估**”。
- **初学者学习提示**：把你常用 Prompt 拆成模块（角色、任务、约束、输出格式），分别测试不同模型，记录差异。

## 2) 用 EDD（评估驱动开发）提升 LLM 可靠性
- **来源**：Planet AI Aggregate（LangChain）
- **链接**：https://www.langchain.com/blog/iterating-towards-llm-reliability-with-evaluation-driven-development
- **发生了什么**：文章分享如何通过评估驱动开发和生产监控，持续迭代 LLM 产品可靠性。
- **为什么重要**：从“我觉得答案不错”升级到“有指标地改进”，这是训练师职业化的关键。
- **初学者学习提示**：先定义 3 个基础指标：正确率、格式合规率、拒答合规率。每次改 Prompt 都跑一遍。

## 3) Agent 评估就绪清单发布
- **来源**：Planet AI Aggregate（LangChain）
- **链接**：https://www.langchain.com/blog/agent-evaluation-readiness-checklist
- **发生了什么**：提供 Agent 评估清单，覆盖错误分析、数据集构建、评分器设计、离线/在线评估和上线准备。
- **为什么重要**：Agent 比普通问答复杂，失败点更多（工具调用、步骤中断、状态错乱）。
- **初学者学习提示**：先从“失败样本库”开始：每周收集 20 条失败轨迹，比盲目加数据更有效。

## 4) OpenEvals：快速启动 LLM 与 Agent 评估
- **来源**：Planet AI Aggregate（LangChain）
- **链接**：https://www.langchain.com/blog/evaluating-llms-with-openevals
- **发生了什么**：推出可快速使用的评估器，支持 LLM-as-a-judge、结构化输出和 Agent 轨迹评估。
- **为什么重要**：降低了评估门槛，让中小团队也能做“持续评估”。
- **初学者学习提示**：优先用“结构化输出评估”（如 JSON 字段完整性）入门，主观评分后置。

## 5) Sun Finance：ID 提取与反欺诈生成式 AI 落地
- **来源**：Planet AI Aggregate（AWS）
- **链接**：https://aws.amazon.com/blogs/machine-learning/sun-finance-automates-id-extraction-and-fraud-detection-with-generative-ai-on-aws/

## 今日概念补充

- **EDD（Evaluation-Driven Development）**：先定评估标准，再改系统。避免“改了很多但不知道是否更好”。
- **LLM-as-a-Judge**：用一个模型给另一个模型打分，适合规模化初筛，但要配合人工抽检。
- **离线评估 vs 在线评估**：离线在固定数据集跑；在线看真实用户环境表现。
- **模型可迁移性（Model Agility）**：同一业务流程可切换不同模型，降低供应风险。
- **Agent 轨迹（Trajectory）**：Agent 从输入到输出的全步骤记录，是调试与评估核心材料。

### 3) 简单 Python 评估片段（字段级准确率）

```python
def field_accuracy(pred, gold, fields):
    hit = 0
    for f in fields:
        if pred.get(f) == gold.get(f):
            hit += 1
    return hit / len(fields)

pred = {"name":"张三","id_number":"110101199001011234","birth_date":"1990-01-01","address":"北京市朝阳区"}
gold = {"name":"张三","id_number":"110101199001011234","birth_date":"1990-01-01","address":"北京市朝阳区"}

score = field_accuracy(pred, gold, ["name","id_number","birth_date","address"])
print("字段准确率:", score)
```
