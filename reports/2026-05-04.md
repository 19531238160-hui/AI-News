- **发生了什么**：文章总结了把提示词从“经验法”升级为“工程法”的方法，包括负约束（明确禁止内容）、结构化 JSON 输出、以及多候选推理采样。

### 6) 什么是 Tokenization Drift（分词漂移）以及如何修复
- **来源**：MarkTechPost（Planet AI Aggregate）
- **链接**：https://www.marktechpost.com/2026/05/03/what-is-tokenization-drift-and-how-to-fix-it/
- **发生了什么**：文章指出模型表现波动可能来自输入格式细节变化（空格、换行、标点）导致分词不同。
- **为什么重要**：训练师常误以为是“模型变差”，其实可能是预处理格式不一致。
- **初学者学习提示**：建立“输入标准化”流程：统一换行、空格、Unicode 规范，再做评测。

## 今日概念补充

- **负约束（Negative Constraints）**：在提示词里明确“不要做什么”，用于减少越界输出。  
- **结构化输出（Structured Output）**：要求模型按 JSON/表格返回，便于自动评估与下游系统调用。  
- **多假设采样（Multi-hypothesis Sampling）**：一次生成多个候选，再用规则或模型重排。  
- **分词漂移（Tokenization Drift）**：输入格式轻微变化引起 token 序列变化，导致输出质量波动。  
- **任务级评估（Task-level Eval）**：不是只看单题对错，而是看整条工作流是否完成目标。

## 代码示例

### 1) 提示词模板（结构化 + 负约束）

```text
你是一个信息抽取助手。
任务：从用户文本中抽取事件信息。

硬性要求：
1) 仅输出合法 JSON，不要输出任何解释。
2) 不要编造不存在的实体；无法确定时填 null。
3) 时间统一为 ISO-8601 格式。

JSON Schema:
{
  "event": "string",
  "date": "string|null",
  "location": "string|null",
  "participants": ["string"],
  "confidence": "number(0-1)"
}
```

### 2) 数据标注 JSON（训练师常用）

```json
{
  "id": "sample_001",
  "input": "明天下午三点在上海张江开产品评审会，李雷和韩梅梅参加。",
  "label": {
    "event": "产品评审会",
    "date": null,
    "location": "上海张江",
    "participants": ["李雷", "韩梅梅"]
  },
  "meta": {
    "annotator": "trainer_a01",
    "copyright_status": "internal_authorized",
    "quality_check": "pass"
  }
}
```
