### 1) Sakana AI 发布 KAME 语音架构（实时注入 LLM 知识）
- **来源**：MarkTechPost（Planet AI Aggregate）  
- **链接**：https://www.marktechpost.com/2026/05/03/sakana-ai-introduces-kame-a-tandem-speech-to-speech-architecture-that-injects-llm-knowledge-in-real-time/  
- **发生了什么**：报道称 Sakana AI 提出 KAME，一种串联式语音到语音架构，可在对话过程中实时引入 LLM 知识，同时强调不额外增加延迟。

- **初学者学习提示**：关注三个评估维度：**延迟（ms）/事实准确率/对话自然度**，不要只看单一分数。

- **初学者学习提示**：先画出 Agent 职责图（输入、输出、依赖），再定义每个节点的质量检查规则（如证据来源、置信度阈值）。

### 3) 什么是 Tokenization Drift，如何修复
- **来源**：MarkTechPost（Planet AI Aggregate）  
- **链接**：https://www.marktechpost.com/2026/05/03/what-is-tokenization-drift-and-how-to-fix-it/  
- **发生了什么**：文章指出模型性能波动可能并非数据变了，而是输入格式细节（空格、换行、标点）导致分词结果改变。  
- **为什么重要**：这是训练与评测常见“暗坑”。同一语义输入，token 序列不同，可能触发输出质量下降。  
- **初学者学习提示**：建立**输入规范化（normalization）**步骤，并在评估日志里记录 token 数、模板版本、预处理版本号。

## 今日概念补充

- **Tokenization Drift（分词漂移）**：文本看似没变，但分词结果变了，导致模型输出波动。  
- **Test-Time Compute（推理时计算）**：模型在回答阶段消耗的计算资源；越“深思熟虑”通常越贵越慢。  
- **Agentic Workflow（智能体编排）**：把复杂任务拆成多个角色协作，每个角色有专门工具与职责。  
- **SWE-Bench Verified**：面向代码修复/工程任务的一类评测指标，常用于比较模型工程能力。

## 代码示例

### 1) 提示词模板（训练师用于稳定输出）

```text
你是{角色}。请基于{输入文本}完成{任务目标}。
约束：
1) 使用{输出格式}；
2) 不要编造未提供事实；
3) 若信息不足，明确写“信息不足”；
4) 输出前先自检：是否满足格式、是否包含来源字段。
```

### 3) 简单 Python 片段（统计推理成本）

```python
records = [
    {"task": "qa_1", "in_tokens": 320, "out_tokens": 180, "latency_ms": 2400},
    {"task": "qa_2", "in_tokens": 500, "out_tokens": 260, "latency_ms": 4100},
]

avg_latency = sum(r["latency_ms"] for r in records) / len(records)
avg_tokens = sum(r["in_tokens"] + r["out_tokens"] for r in records) / len(records)

print(f"平均时延: {avg_latency:.1f} ms")
print(f"平均总token: {avg_tokens:.1f}")
```
