## 重点新闻（9 条）

### 1) Fastino 开源 GLiGuard（300M 安全审核模型）
- **来源**：MarkTechPost（聚合源：Planet AI Aggregate）  
- **链接**：https://www.marktechpost.com/2026/05/13/fastino-labs-open-sources-gliguard-a-300m-parameter-safety-moderation-model-that-matches-or-exceeds-accuracy-of-models-23-90x-its-size/  
- **发生了什么**：Fastino Labs 发布开源安全审核模型 GLiGuard，覆盖提示词安全、越狱策略检测、危害分类、拒答检测等任务，主打单次前向推理完成多任务。  
- **为什么重要**：如果轻量模型能兼顾效果与吞吐，会显著降低安全审核成本，适合高并发应用。  
- **初学者学习提示**：先学会把“安全”拆成可标注子任务（如辱骂、违法、越狱、隐私），再做多标签评估，而不是只看“是否违规”一个标签。

### 2) 生产级 Agent 的 12 指标评估框架
- **来源**：Towards Data Science（聚合源）  
- **链接**：https://towardsdatascience.com/building-an-evaluation-harness-for-production-ai-agents-a-12-metric-framework-from-100-deployments/  
- **发生了什么**：文章总结了来自 100+ 部署经验的 12 项评估指标，覆盖检索、生成、Agent 行为和生产健康。  
- **为什么重要**：训练师常见误区是只看“回答像不像”，忽视延迟、失败率、工具调用成功率等线上指标。  
- **初学者学习提示**：把你的评估分成四栏：**质量、行为、成本、稳定性**，每栏至少 2 个可量化指标。

- **来源**：LangChain 官方博客（聚合源）  
- **链接**：https://www.langchain.com/blog/introducing-llm-gateway

### 5) Nous 发布 TST：预训练提速最高 2.5x
- **来源**：MarkTechPost（聚合源）  
- **链接**：https://www.marktechpost.com/2026/05/13/nous-research-releases-token-superposition-training-to-speed-up-llm-pre-training-by-up-to-2-5x-across-270m-to-10b-parameter-models/  
- **发生了什么**：Nous 提出 Token Superposition Training（TST），两阶段训练以缩短训练墙钟时间，并声称不改推理时行为。  
- **为什么重要**：训练效率提升意味着更快迭代、更低成本，也可能带来更多“小步快跑”的模型实验。  
- **初学者学习提示**：先理解两个概念：**FLOPs（理论算力开销）** 与 **wall-clock（实际训练耗时）** 不总是同向。

### 7) Inference Routing：按任务分配模型
- **来源**：DigitalOcean 教程（聚合源）  
- **链接**：https://www.digitalocean.com/community/tutorials/inference-routing-model-task-matching  
- **发生了什么**：介绍了推理路由思路：根据任务类型而非固定 API 调用单一模型，平衡质量与成本。  
- **为什么重要**：很多业务并不需要全用最贵模型；路由策略能显著降本。  
- **初学者学习提示**：从最简单的“规则路由”开始：摘要→小模型，复杂推理→大模型，敏感内容→安全模型。

## 今日概念补充

- **Guardrail（护栏）**：限制模型输出风险的规则与模型组合。  
- **PII 脱敏**：对身份证号、手机号、邮箱等个人信息做掩码或替换。  
- **Trace 连续性**：一次用户请求在多模型、多工具链路中的可追踪记录。  
- **Inference Routing**：根据任务特征动态选择模型。  
- **Wall-clock Time**：真实世界的训练耗时，不等同理论计算量。
