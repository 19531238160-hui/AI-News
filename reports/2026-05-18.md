## 今日概念补充

- **MoE（Mixture of Experts）**：把模型分成多个“专家子网络”，每次只激活部分专家，提升效率。  
- **自回归（AR） vs 扩散（Diffusion）**：AR 顺序生成；扩散通过迭代 refinement 生成，潜在并行度更高。  
- **评估层（Evaluation Layer）**：连接“模型输出”与“上线决策”的规则系统，强调可复现与可审计。  
- **会话持久化（Session Persistence）**：系统重启或多轮交互后仍保留上下文状态。  
- **SHAP**：用博弈论思想估计特征对预测结果的贡献，常用于解释黑盒模型。

## 代码示例（AI 训练师向）

```python
# 一个极简“可上线判定”评估器：把主观感觉拆成可复现分数
# 维度：factuality(事实性), relevance(相关性), specificity(具体性), safety(安全性)

from dataclasses import dataclass

@dataclass
class EvalScore:
    factuality: float   # 0~1
    relevance: float
    specificity: float
    safety: float

def ship_decision(s: EvalScore) -> dict:
    # 可按业务调整权重与阈值
    weighted = (
        0.35 * s.factuality +
        0.25 * s.relevance +
        0.20 * s.specificity +
        0.20 * s.safety
    )
    hard_fail = (s.safety < 0.85) or (s.factuality < 0.75)
    decision = "BLOCK" if hard_fail else ("SHIP" if weighted >= 0.82 else "REVIEW")
    return {"score": round(weighted, 3), "decision": decision}

example = EvalScore(factuality=0.80, relevance=0.88, specificity=0.70, safety=0.92)
print(ship_decision(example))
# 输出示例: {'score': 0.823, 'decision': 'SHIP'}
```

> 训练师实操建议：先定义“硬性否决条件”（如安全<0.85），再定义“加权总分”，最后对灰区样本走人工复审。
