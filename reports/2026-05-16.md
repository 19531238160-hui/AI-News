## 今日概念补充

- **MoE（Mixture of Experts）**：把模型分成多个“专家子网络”，按需激活一部分，提升效率。  
- **自回归（AR）vs 扩散（Diffusion）**：AR 逐 token 生成；扩散通常通过迭代去噪生成，二者在速度/质量/硬件适配上取舍不同。  
- **Benchmark 污染**：评测题可能被模型训练见过，导致成绩虚高。  
- **Harness（推理编排框架）**：围绕模型的一整套运行策略（提示模板、工具链、重试、验证）。  
- **Checkpointing**：保存中间状态，便于失败恢复与长任务续跑。

## 代码示例

### 1) 提示词模板：强制语言一致性（适合多语言助手评测）
```text
你是代码助理。请严格遵循：
1) 始终使用【简体中文】回答；
2) 代码注释也使用【简体中文】；
3) 若你想切换语言，先说明理由并等待用户确认；
4) 输出格式：
   - 问题理解
   - 解决步骤
   - 代码
   - 自检清单（语言一致性、可运行性、边界条件）
```

### 2) 数据标注 JSON：Agent 任务结果记录
```json
{
  "task_id": "agent_eval_20260516_001",
  "user_query": "请用Python读取CSV并统计每列缺失值",
  "expected_language": "zh-CN",
  "rubric": {
    "correctness": 0.4,
    "tool_use": 0.2,
    "language_consistency": 0.2,
    "safety": 0.2
  },
  "model_output": "...",
  "scores": {
    "correctness": 4,
    "tool_use": 5,
    "language_consistency": 3,
    "safety": 5
  },
  "final_score": 4.2,
  "notes": "内容正确，但夹杂韩文术语，语言一致性扣分"
}
```

### 3) 简单 Python：计算加权评估分
```python
rubric = {"correctness": 0.4, "tool_use": 0.2, "language_consistency": 0.2, "safety": 0.2}
scores = {"correctness": 4, "tool_use": 5, "language_consistency": 3, "safety": 5}  # 1-5分

final = sum(rubric[k] * scores[k] for k in rubric)
print(f"Final Score: {final:.2f}")
```
