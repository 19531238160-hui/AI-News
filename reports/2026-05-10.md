# AI 训练师学习型新闻简报（2026-05-10）

> 时间窗口：最近 24–48 小时  
> 来源策略：RSS / 公开源

## 今日速览

- **LLM 工程基础能力图谱**再次被强调：从分词、上下文窗口到评估框架，训练师需要“端到端理解”而不只是会写提示词。  
- **Agent 安全**成为高频主题：当模型接入工具、记忆和浏览器会话后，攻击面明显扩大。  
- **RAG 进入“时间敏感”阶段**：只看语义相似度会导致“过时但看起来正确”的答案。  
- **AI 编码 Agent 工程化**在加速：规范驱动（spec-driven）开发、浏览器内执行、多步骤工作流成为新实践。  
- **模型效率与架构创新**继续推进：如单 checkpoint 多尺寸推理模型、MoE 预训练模块化。  
- **职业能力迁移**：从“做模型”到“做系统”，AI 训练师需要理解产品、评估、安全和协作流程。

### 2) AI Agent 的安全攻击面：工具与记忆带来的新风险
- **来源**: Planet AI Aggregate（转载 Towards Data Science）
- **链接**: https://towardsdatascience.com/the-ai-agent-security-surface-what-gets-exposed-when-you-add-tools-and-memory/
- **发生了什么**: 文章提出了 Agent 工作流在后端暴露面的结构化安全分析框架。
- **为什么重要**: 训练师不仅要关注“提示词注入”，还要关注工具调用权限、记忆污染、数据泄露链路。
- **初学者学习提示**: 给每个工具加“最小权限”标签（只读/可写/可执行），在标注数据中加入“是否涉及敏感操作”的字段。

### 4) GitHub Spec-Kit：面向 AI 编码 Agent 的规范驱动开发
- **来源**: Planet AI Aggregate（MarkTechPost）
- **链接**: https://www.marktechpost.com/2026/05/08/meet-github-spec-kit-an-open-source-toolkit-for-spec-driven-development-with-ai-coding-agents/

- **来源**: Planet AI Aggregate（MarkTechPost）
- **链接**: https://www.marktechpost.com/2026/05/09/nvidia-ai-releases-star-elastic-one-checkpoint-that-contains-30b-23b-and-12b-reasoning-models-with-zero-shot-slicing/
- **发生了什么**: 报道称该方法在一个 checkpoint 中嵌入 30B/23B/12B 多个推理规模，降低重复训练成本。
- **为什么重要**: 对部署方意味着可按延迟/成本动态选模型规模；对训练师意味着评测要覆盖多预算场景。

### 6) RAG 的时间盲区：生产环境加入 Temporal Layer
- **来源**: Planet AI Aggregate（Towards Data Science）
- **链接**: https://towardsdatascience.com/rag-is-blind-to-time-i-built-a-temporal-layer-to-fix-it-in-production/

## 今日概念补充

- **Prompt Injection（提示词注入）**：通过恶意文本诱导模型越权。  
- **Agent Attack Surface（攻击面）**：系统所有可被利用的入口集合（提示、工具、记忆、外部 API）。  
- **Spec-driven Development（规范驱动开发）**：先定义需求/验收标准，再生成实现。  
- **Temporal RAG（时序增强检索）**：把时间信息加入检索过滤与重排，减少过时答案。  
- **MoE（专家混合）**：用路由机制只激活部分参数，提升计算效率。

## 代码示例

### 1) 训练师常用提示词模板（规范驱动）

```text
你是资深代码助手。请严格按以下步骤输出：

1. 需求复述（不超过5条）
2. 验收标准（Given-When-Then，至少3条）
3. 实现方案（模块/函数/输入输出）
4. 风险与边界（权限、异常、回滚）
5. 代码实现
6. 自测用例（最少3个）

约束：
- 若需求不明确，先提出澄清问题，不要直接写代码。
- 禁止假设未给出的外部依赖。
```

### 2) 数据标注 JSON（含时效与风险字段）

```json
{
  "query": "2026年最新报销政策是什么？",
  "retrieved_docs": [
    {"id": "d1", "title": "报销政策-2024版", "doc_date": "2024-03-01"},
    {"id": "d2", "title": "报销政策-2026版", "doc_date": "2026-04-20"}
  ],
  "labels": {
    "is_time_sensitive": true,
    "best_doc_id": "d2",
    "is_latest_required": true,
    "risk_level": "high"
  }
}
```
