## 1) Deep Agents 增加“模型专用配置”，强调跨模型调优
- **来源**：Planet AI Aggregate（LangChain 博客）
- **链接**：https://www.langchain.com/blog/tuning-deep-agents-different-models
- **发生了什么**：LangChain 介绍 Deep Agents 从“通用配置”升级为“模型专用 profile（提示词、工具、中间件）”，并称在部分 tau2-bench 子集上较默认方案提升约 10–20 分。  
- **为什么重要**：这说明 **Agent 性能不只靠底座模型，还强依赖编排层配置**。同样任务在不同模型上，提示风格、工具调用策略可能完全不同。  
- **初学者学习提示**：把“换模型”当成一次小迁移项目：  
  1) 固定任务集；2) 分模型做 prompt/tool 配置；3) 记录成功率、成本、延迟三指标。

## 2) RAG 为何在生产失败：从 Demo 到线上的断层
- **来源**：Planet AI Aggregate（DigitalOcean 教程）
- **链接**：https://www.digitalocean.com/community/tutorials/why-rag-systems-fail-in-production
- **发生了什么**：文章总结 RAG 常见失败点：检索差、切块不合理、知识库过期、缺少系统评估。  
- **为什么重要**：很多团队“模型能答”但“业务不可用”，核心不是生成模型本身，而是检索与评估工程。  
- **初学者学习提示**：先做最小评估闭环：  
  - Top-k 命中率（检索）  
  - 答案事实一致性（生成）  
  - 更新时间滞后（数据新鲜度）

## 今日概念补充

- **Model Profile（模型配置档）**：针对不同底座模型的提示词、工具调用、超参、后处理规则集合。  
- **RAG 评估闭环**：检索评估 + 生成评估 + 线上反馈，不是只看最终回答。  
- **PII（个人可识别信息）**：能识别个人身份的信息（邮箱、电话、证件号等），训练前需脱敏。  
- **数据新鲜度（Freshness）**：知识库更新与业务事实的时差，直接影响回答可信度。  
- **回归测试集**：每次改提示词/模型后都要复跑的一组固定样本，用于防止性能倒退。

### 1) 提示词模板（用于多模型 Agent 对比）

输出格式(JSON)：

### 3) 简单 Python 评估片段（RAG 命中率）
```python
def hit_rate(retrieved_docs, gold_doc_id):
    # retrieved_docs: List[List[str]] 每条样本的检索doc_id列表
    # gold_doc_id: List[str] 每条样本的标准答案doc_id
    hit = 0
    for docs, gold in zip(retrieved_docs, gold_doc_id):
        if gold in docs:
            hit += 1
    return hit / len(gold_doc_id)

# 示例
retrieved = [["d1","d3"], ["d2","d5"], ["d9","d8"]]
gold = ["d3","d7","d8"]
print("Top-k HitRate:", hit_rate(retrieved, gold))  # 0.666...
```
