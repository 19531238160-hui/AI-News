## 今日概念补充

- **多模态（Multimodal）**：模型同时处理文本、语音、图像/视频。  
- **MoE（混合专家）**：像“分科医生”，不同子网络处理不同类型问题。  
- **Micro-turn（微回合）**：把交互切成极短时间片，降低对话延迟与打断成本。  
- **Spec-driven（规格驱动）**：先定义需求与验收，再调用模型生成。  
- **FLOPs 跟踪**：记录训练/微调算力消耗，用于合规、成本和复现实验。

## 代码示例

### 1) 提示词模板（规格驱动）

```text
你是AI助手，请根据以下规格完成任务：

[任务目标]
- 输出一段客服回复，解决用户账单争议

[输入]
- 用户消息: {user_text}
- 订单信息: {order_info}

[约束]
- 不编造政策
- 必须引用订单字段
- 100字以内
- 语气：专业、同理心

[输出格式(JSON)]
{
  "reply": "...",
  "evidence": ["订单号", "扣费时间", "套餐名称"],
  "risk_flag": "low|medium|high"
}
```

### 2) 训练数据标注 JSON（可审计字段）

```json
{
  "id": "case_00124",
  "input": {
    "user_query": "为什么我被重复扣费？",
    "context": "订单A在2026-05-01和2026-05-02各扣费一次"
  },
  "label": {
    "intent": "billing_dispute",
    "response_quality": 4,
    "factuality": "supported",
    "evidence_spans": ["2026-05-01", "2026-05-02"],
    "safety": "pass"
  },
  "meta": {
    "annotator": "trainer_07",
    "policy_version": "v1.3"
  }
}
```
