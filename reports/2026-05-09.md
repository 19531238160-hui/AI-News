## 2) AI Agent 安全面正在扩大：工具与记忆带来新攻击面
- **来源**：Towards Data Science（聚合）  
- **链接**：https://towardsdatascience.com/the-ai-agent-security-surface-what-gets-exposed-when-you-add-tools-and-memory/  
- **发生了什么**：文章提出，Agent 一旦接入外部工具和长期记忆，攻击面从 prompt 扩展到后端系统、数据通道与权限链。  
- **为什么重要**：很多团队把安全理解成“防注入”，但真实风险常在**工具执行层**（如 API 调用、文件系统、身份会话）。  
- **初学者学习提示**：做评估时把风险拆成四层：**输入层、规划层、执行层、记忆层**，逐层设计测试样例。

## 今日概念补充

- **Agent（代理）**：能“思考+调用工具+执行动作”的系统，不只是文本生成器。  
- **攻击面（Attack Surface）**：系统可能被利用的入口总和。工具越多、权限越高，攻击面通常越大。  
- **规格驱动开发（Spec-Driven）**：先定义需求与验收标准，再让 AI 生成和迭代。  
- **遥测（Telemetry）**：运行时日志与行为信号，用于审计、告警和故障排查。  
- **登录态风险**：AI 若可使用用户已登录会话，误操作可能直接影响真实业务数据。

### 2) 提示词模板（规格驱动）

```text
你是企业内部自动化助手。请严格按以下规格执行：

[目标]
- 生成“周报汇总”草稿，不执行发送动作。

[输入]
- 数据源：本周工单 CSV（只读）
- 输出格式：Markdown

[验收标准]
1. 必须包含：完成项/风险项/下周计划 三部分
2. 不得包含客户手机号、邮箱等敏感信息
3. 若数据缺失，明确标注“待补充”，不得编造

[安全约束]
- 禁止调用发送邮件、外部上传、删除文件等工具
- 任何越权请求需先回复“需要人工审批”
```
