## 重点新闻（10 条）

## 1) Open Agent Leaderboard 发布
- **来源**：Hugging Face Blog  
- **链接**：https://huggingface.co/blog/ibm-research/open-agent-leaderboard  
- **发生了什么**：发布面向开放智能体（Agent）的排行榜，用统一任务与指标对不同 Agent 方案进行比较。  
- **为什么重要**：  
  - Agent 不再只看“会不会聊天”，而是看**任务完成率、稳定性、工具调用质量**。  
  - 对训练师来说，评测标准会直接影响数据构造与提示词策略。  
- **初学者学习提示**：先学会区分**模型能力评测**（如问答）与**Agent 评测**（多步任务+工具调用）。

## 2) OpenAI 推进内容溯源与可验证媒体
- **来源**：OpenAI Blog  
- **链接**：https://openai.com/index/advancing-content-provenance  
- **发生了什么**：提出并推进内容溯源方案，包括 Content Credentials、SynthID 与验证工具，帮助识别 AI 生成媒体。  
- **为什么重要**：  
  - 训练数据治理与平台信任会越来越依赖“来源可追踪”。  
  - 在审核、标注、风控场景，可验证信号将成为关键元数据。  
- **初学者学习提示**：标注时新增“**来源字段**”“**是否疑似合成**”“**证据链**”三类字段，提升数据可审计性。

## 3) OpenAI 与 Dell 合作，推进 Codex 企业混合部署
- **来源**：OpenAI Blog  
- **链接**：https://openai.com/index/dell-codex-enterprise-partnership  
- **发生了什么**：双方推动 Codex 在混合云与本地环境部署，面向企业代码工作流与数据安全需求。  
- **为什么重要**：  
  - 企业更在意权限、审计、合规，而非单纯“模型是否更大”。  
  - AI 训练师将更多参与“任务分级、提示模板标准化、输出审核策略”。  
- **初学者学习提示**：学习设计“**可审计提示词模板**”（含任务、约束、日志字段）。

## 4) Project Genie + Street View：真实地点模拟能力
- **来源**：Google DeepMind Blog  
- **链接**：https://deepmind.google/blog/simulate-real-world-places-with-project-genie-and-street-view/  
- **发生了什么**：引入基于 Street View 的真实地点模拟能力。  
- **为什么重要**：  
  - 这类世界模型能力可用于机器人、导航、空间理解训练。  
  - 训练师可参与“场景描述数据”“行为评估标准”建设。  
- **初学者学习提示**：关注“**场景-动作-结果**”三元标注法，适用于仿真任务数据集。

## 5) Ettin Reranker 系列发布
- **来源**：Hugging Face Blog  
- **链接**：https://huggingface.co/blog/ettin-reranker  
- **发生了什么**：发布新 Reranker（重排模型）家族。  
- **为什么重要**：  
  - 在 RAG 中，重排常常比“再换一个大模型”更省成本且更有效。  
  - 训练师可通过难例构造显著提升检索质量。  
- **初学者学习提示**：先做一个小评测：Top-k 检索后加 Reranker，比对命中率/正确率变化。

## 6) Co-Scientist 用于加速细胞年轻化线索发现

- **链接**：https://deepmind.google/blog/fast-tracking-genetic-leads-to-reverse-cellular-aging/

## 7) NVIDIA Cosmos Predict 2.5：LoRA/DoRA 机器人视频微调

## 今日概念补充

- **Content Provenance（内容溯源）**：记录内容来源、生成与编辑链路，帮助判断可信度。  
- **Reranker（重排模型）**：对初检索结果二次排序，常用于提升 RAG 准确率。  
- **LoRA/DoRA**：参数高效微调方法，用较少训练参数实现任务适配。  
- **Agent 评测**：不仅看答案质量，还看规划、工具调用、执行成功率与鲁棒性。  
- **混合部署（Hybrid）**：云上+本地协同，常见于合规与数据敏感行业。

### 1) 提示词模板（AI 训练师常用：可审计）
```text
[任务]
你是企业代码助手，请根据需求生成可运行的 Python 函数。

[输入]
- 需求描述: {requirement}

[输出格式]
1. 方案说明（不超过120字）
2. 代码
3. 单元测试
4. 风险与假设（列出不确定点）

[评估标准]
- 正确性
- 可读性
- 安全性
- 是否满足约束
```

### 3) 简单评估表（RAG + Reranker）
```python
samples = [
    {"q": "合同违约金上限", "hit@5_before": 0, "hit@5_after": 1},
    {"q": "发票抬头变更流程", "hit@5_before": 1, "hit@5_after": 1},
]
improve = sum(s["hit@5_after"] - s["hit@5_before"] for s in samples) / len(samples)
print("平均提升:", improve)
```
