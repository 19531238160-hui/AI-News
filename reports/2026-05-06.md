## 今日概念补充

- **默认模型（Default Model）**：平台未特别指定时自动使用的模型。会影响你所有日常产出质量。  
- **幻觉（Hallucination）**：模型生成看似合理但不真实的信息。训练师核心任务之一是降低它。  
- **可观测性（Observability）**：能否看清系统内部行为（日志、链路、调用、错误），便于定位问题。  
- **Agent 路由（Tool Routing）**：根据任务类型把请求分发给不同工具/技能。  
- **评估集（Eval Set）**：固定的一组测试样本，用来比较不同模型/提示词版本。

## 代码示例

### 1) 提示词模板（训练师通用评测）
```text
你是评测助手。请根据“事实一致性、任务完成度、表达清晰度”对回答打分（1-5分），并给出一句改进建议。

【用户问题】
{question}

【模型回答】
{answer}

【参考事实（可为空）】
{reference}

输出JSON：
{
  "factuality": 1-5,
  "task_completion": 1-5,
  "clarity": 1-5,
  "suggestion": "..."
}
```

### 2) 数据标注 JSON 样例（适合做 SFT / Eval）
```json
{
  "id": "news_eval_20260506_001",
  "instruction": "总结新闻并说明对AI训练师的意义",
  "input": "GPT-5.5 Instant 成为默认模型...",
  "output": {
    "summary": "新模型成为默认选项，强调准确性和低幻觉。",
    "trainer_value": "需更新评测基线并记录模型版本。"
  },
  "labels": {
    "domain": "llm_product_update",
    "difficulty": "beginner",
    "risk_flags": ["possible_hallucination"]
  }
}
```

### 3) 简单 Python：计算评测平均分
```python
scores = [
    {"factuality": 4, "task_completion": 5, "clarity": 4},
    {"factuality": 3, "task_completion": 4, "clarity": 4},
    {"factuality": 5, "task_completion": 4, "clarity": 5},
]

avg = {k: sum(d[k] for d in scores)/len(scores) for k in scores[0]}
print(avg)
# {'factuality': 4.0, 'task_completion': 4.33, 'clarity': 4.33}
```
